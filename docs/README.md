# docs/

## Purpose

All project documentation lives here. This is the first place to look before making any
change to the codebase.

## Structure

- [`DESIGN/`](DESIGN/) — High-level and low-level design documents. Read these before
  touching any code. The HLD is the authoritative specification.
- [`BACKLOG/`](BACKLOG/) — Epics and user stories. One folder per epic. Check here for
  current implementation status and what to work on next.
- [`WORKLOGS/`](WORKLOGS/) — Session-by-session progress log. Read the last few entries
  before starting a session to understand current state and pick up from documented next steps.

## Rules

- Read the latest worklog entry before starting any work
- Read the relevant LLD before touching any component
- Update backlog story checklists as you complete tasks
- Write a worklog entry before ending any meaningful session
- Never leave the WORKLOGS/README.md index out of date
